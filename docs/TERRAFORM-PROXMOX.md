# Terraform Proxmox Integration

## Purpose
This document explains how the Terraform stack is wired into the repo and how it consumes secrets generated by the existing Task workflows.

## Secret flow
The Terraform tasks do not keep credentials in `.tf` or `.tfvars` files.
They source the existing repo-managed files below instead:

- `state/configs/.env`
- `state/secrets/proxmox/proxmox-api-token.env`
- `state/secrets/ssh/id_ed25519_homelab`
- `state/secrets/ssh/id_ed25519_homelab.pub`

These files are already part of the current bootstrap pattern.

## Required bootstrap order
1. `task init`
2. `task secrets:ssh:keys`
3. `task proxmox:env:ensure`
4. `task proxmox:api-user`
5. `task terraform:init`
6. `task terraform:validate`
7. `task terraform:plan`

## Notes
- The Terraform stack uses the `bpg/proxmox` provider.
- Ubuntu 24.04 and 25.10 downloads are enabled by default.
- Ubuntu 26.04 remains disabled by default until the final release media is confirmed.
- Ubuntu LXC templates are downloaded with `proxmox_virtual_environment_download_file` using `content_type = "vztmpl"`.
- VM cloud images are also downloaded before template VMs are created.

## Recommended `.env` additions
Add the following keys to `state/configs/.env` if you want explicit datastore and node overrides:

```dotenv
PROXMOX_NODE_NAME=pve01
PROXMOX_INSECURE_TLS=true
PROXMOX_TEMPLATE_DATASTORE=local
PROXMOX_SNIPPETS_DATASTORE=local
PROXMOX_VM_DATASTORE=local-lvm
```
